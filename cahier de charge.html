Cahier des Charges Global : Système d'Information Intégré & BI
(Organisme "Info 1")

1. Contexte et Objectifs du Projet

L'organisme "Info 1" souhaite moderniser et digitaliser la gestion de ses 6 départements clés. L'objectif est double :

  • Opérationnel : Fournir à chaque département une application web sur-mesure pour gérer ses activités quotidiennes.
  • Stratégique (Data) : Permettre aux responsables de chaque département d'analyser leurs propres données, et fournir à la Direction Générale (le "Chef") une vue consolidée et globale de l'organisme via des tableaux de bord interactifs.

2. Architecture Globale du Système

Le système repose sur une architecture décentralisée pour la collecte des données et centralisée pour l'analyse décisionnelle.

  • Couche Opérationnelle (Web) : 6 applications web indépendantes, développées et gérées par 6 équipes distinctes.
  • Couche Persistance (Bases de données) : Utilisation délibérée de bases de données hétérogènes (MySQL, PostgreSQL, MongoDB) selon les besoins structurels de chaque département pour simuler un environnement d'entreprise réaliste.
  • Couche Analytique Locale (Département) : Une instance Metabase connectée à la base de données de chaque application pour les besoins analytiques spécifiques du département.
  • Couche Décisionnelle Globale (Direction) : Une instance Metabase centrale, connectée simultanément aux 6 bases de données, permettant de croiser les informations.

3. Répartition des Équipes et Domaines

Le projet est divisé entre 6 groupes de 6 personnes. Chaque groupe est totalement responsable du cycle de vie de son application (conception, développement, déploiement, analyse).

Groupe Département ciblé Piste de Base de Données Suggérée
Groupe 1 Ressources Humaines (RH) PostgreSQL (Idéal pour les données relationnelles)
Groupe 2 Instruction (Notes, Examens) PostgreSQL ou MySQL
Groupe 3 Restauration (Menus, Caisses) MongoDB ou MySQL (Système autonome)
Groupe 4 Trésorerie (Budget, Flux) MySQL (Robuste pour les transactions financières)
Groupe 5 Matériel (Inventaire, Prêts) MongoDB ou MySQL
Groupe 6 Gestion de Courrier (Entrant/Sortant) MongoDB (Stockage orienté document pertinent)

4. Spécifications Techniques Globales

Pour assurer la cohérence et la réussite de la phase d'intégration finale, toutes les équipes doivent respecter les standards suivants :

4.1. Développement Web

  • Stack Technologique : Le choix du langage Frontend et Backend est libre pour chaque équipe.
  • Gestion de Version : L'utilisation de Git est strictement obligatoire (avec utilisation de branches dédiées par fonctionnalité).

4.2. Bases de Données & Data Ingénierie

  • Modélisation : Chaque groupe doit produire un dictionnaire de données et un schéma de base de données clair.
  • Gouvernance des Données : Standardisation des clés primaires. L'identifiant "Matricule" doit avoir un format commun entre les groupes pour permettre les futures jointures sur la BI globale.
  • Peuplement (Mock Data) : Chaque groupe doit concevoir un script pour générer un jeu de données fictives volumineux afin de rendre la data analyse pertinente.

5. Volet Data Analyst & Business Intelligence (Metabase)

La valorisation de la donnée est le cœur de ce projet d'ingénierie.

  • Niveau 1 : Tableaux de bord locaux (par groupe)
    Chaque application doit comporter des KPIs spécifiques à son domaine (ex: Taux d'absentéisme pour les RH).
  • Niveau 2 : Le Tableau de bord "Chef" (Global)
    Connexion de l'instance Metabase "Direction" aux 6 bases. Création de KPIs transverses (ex: Croiser le budget alloué par la Trésorerie avec les dépenses du département Matériel).

6. Livrables Attendus (Sprint de 16h)

À la fin du projet (4 séances de 4h), chaque groupe devra fournir :
  1. Le lien du dépôt Git avec l'historique complet de l'équipe.
  2. Le script de création et de peuplement massif de la base de données.
  3. L'application Web fonctionnelle permettant la saisie rapide des données.
  4. Le tableau de bord Metabase local exporté.
